# 마이데이터 (본인신용정보관리업)

## 정의
**마이데이터**는 정보주체 본인이 여러 금융회사 등에 흩어져 있는 본인의 신용정보를 한곳에서 통합 조회·관리할 수 있도록 하는 서비스이다. 법적으로는 **본인신용정보관리업**이라 하며, 근거법은 **신용정보법**이다. 사업을 영위하려면 **금융위원회**의 허가를 받아야 하고, 검사·감독은 **금융감독원**이 수행한다.

## 데이터 전송 방식 (API)
마이데이터는 스크래핑을 금지하고 **표준 API** 방식으로 데이터를 전송한다. 정보제공자(은행·카드사·증권사 등)는 정보주체의 **전송요구**에 따라 본인신용정보관리회사(마이데이터 사업자)에게 신용정보를 API로 제공한다. 전송요구권의 근거는 **신용정보법**의 신용정보 전송요구와 **개인정보 보호법**의 개인정보 이동권이다.

## 보안 요건
- **인증·인가**: 마이데이터 API 접근은 OAuth 2.0 기반의 접근토큰으로 인가한다.
- **접근매체 관리**: 인증수단의 발급·관리는 **전자금융거래법**의 접근매체 규율과 연계된다.
- **정보보호 체계**: 마이데이터 사업자는 **전자금융감독규정**에 준하는 정보보호 통제(망분리, 접근통제, 암호화)를 적용하고, 침해사고 발생 시 **금융보안원**·금융감독원에 보고한다.

## 이용자 보호
마이데이터 사업자는 수집한 신용정보를 전송요구 목적 범위에서만 이용하여야 하며, 정보주체는 언제든지 전송요구를 철회할 수 있다. 개인정보 유출이 발생하면 **개인정보 보호법**에 따른 유출 통지·신고 절차가 적용된다.
