# 데모 시나리오 — Agentic RAG + Knowledge Graph (Chat UI)

watsonx 스택 위 **설명가능한 에이전틱 RAG** 라이브 시연 런북. 무대에서 순서대로 따라 하면 됩니다.
한 줄 메시지: **"질문에 맞는 도구(벡터·그래프·SQL)를 에이전트가 스스로 고르고, 그 근거를 화면에 펼쳐 보여준다."**

- **챗 UI**: https://rag-ui-genai-apps.apps.itz-xe29ld.infra01-lb.tok04.techzone.ibm.com
- **로그인**: 공유 비밀번호(`APP_PASSWORD`, 현장에서 별도 전달)
- **오케스트레이션**: Langflow의 watsonx granite 에이전트(tool-calling) · 장애 시 파이썬 엔진 폴백
- 총 소요: **약 8–10분** (Act 1~4). Act 5(Langflow 시각화)는 선택.

## Act 1 — 세 가지 검색 경로 (라우팅 + 근거) · 약 3분

welcome 화면의 **추천 질문 칩**을 클릭하거나 입력창에 직접 타이핑. 답이 나오면 **상단 tool-trace 배지**(vector/graph/sql 점등)와 **근거 패널 펼치기**를 반드시 보여준다.

### 1-a. vector (의미 검색)
- **질문**: `가명정보란 무엇이고 동의 없이 쓸 수 있는 목적은?`
- **기대**: `vector` 배지 점등 · 답변 + 인용칩 · "Retrieved passages" 패널(청크별 유사도 바)
- **말할 점**: "개념·정의형 질문은 의미 기반 벡터 검색으로 갑니다. 답 아래 실제 검색된 청크와 유사도가 그대로 보입니다 — 블랙박스가 아닙니다."
- **보너스(메타) 질문**: `Docling 문서 포맷은?` (또는 `Langflow는 무엇이고 어떻게 쓰나`)
  - **기대**: `vector` 배지 · Docling/Langflow **README**(URL로 적재된 영문 문서)만 인용 · 실제 지원 포맷(PDF·DOCX·PPTX·HTML…) 나열
  - **말할 점**: "코퍼스엔 한글 법령뿐 아니라 **이 데모가 실제로 쓰는 오픈소스 도구(Docling·Langflow)의 README도 URL로 적재**돼 있습니다 — 시스템이 **자기 스택을 스스로 설명**하죠. 한글 법령 + 영문 기술문서가 한 코퍼스에 섞여도(다국어·이종 소스) 동작합니다."
  - **팁**: 영문 README는 **짧은 키워드형**(`Docling 문서 포맷은?`)이 안정적. 서술형(`어떤 포맷을 지원해?`)은 한글↔영문 임베딩 갭으로 빗나갈 수 있음(그땐 영어로 `What formats does Docling support?`).

### 1-b. graph (관계)
- **질문**: `전자금융거래법의 접근매체와 감독기관의 관계는?`
- **기대**: `graph`(+vector) 배지 · "Knowledge subgraph" 패널(엔티티 칩 + 엣지 트리플)
- **말할 점**: "관계형 질문은 지식그래프로 갑니다. 법–감독기관–제도가 엔티티와 엣지로 펼쳐집니다. AstraDB에 노드/엣지를 NoSQL로 담고, 엔티티 임베딩으로 의미 시드를 찾아 1-홉 이웃을 구성합니다."

### 1-c. sql (text-to-SQL)
- **질문**: `위험등급이 high인 고객의 플래그된 거래 총액을 상대국가별로?`
- **기대**: `sql` 배지 · "Generated SQL · results" 패널(**생성된 SQL 코드블록** + 결과 표; PA/KY/VG 등 고위험국)
- **말할 점**: "집계형 질문은 자연어를 SQL로 바꿔 watsonx.data(Iceberg/Presto)에 실행합니다. granite가 만든 SQL을 가드(읽기 전용)로 검증 후 실행하고, **생성된 쿼리까지 화면에 보여줍니다.** 3개 테이블 조인을 자동으로 만들었죠."

### 1-c′. sql — 문서에서 추출한 정형 데이터 (핵심: "세 번째 형태")
- **질문**: `법별로 의무가 몇 건씩인지 많은 순으로?` (또는 `의무 주체별로 의무 건수는?`)
- **기대**: `sql` 배지 · 생성 SQL(`... iceberg_data.rag.obligations GROUP BY law ...`) + 결과 표(마이데이터 6 · 개인정보보호법 4 …)
- **말할 점**: "방금 SQL은 별도 비즈니스 데이터가 아니라 **적재한 법령 문서에서 granite가 뽑아낸 의무 테이블**입니다. 같은 문서가 벡터·그래프뿐 아니라 **정형 행(SQL)으로도** 저장돼 자연어 집계가 됩니다. text-to-SQL이 질문에 맞춰 AML 데이터셋과 의무 테이블 중 알맞은 쪽을 고릅니다."
- **팁**: "법별/주체별 **건수·집계**" 형태로 물으면 확실히 `sql`로 갑니다. "…의무를 보여줘"처럼 물으면 라우터가 vector로 갈 수 있음(답은 여전히 정확).

> 라우터가 가끔 graph를 vector로만 분류하면, "…를 **지식그래프(엔티티·엣지)로** 보여줘"라고 명시하면 확실히 켜집니다.

## Act 2 — 멀티턴 후속 질문 + 새 대화 · 약 1.5분

같은 대화 안에서 **이전 맥락을 이어받는** 것을 보여준다.

1. **질문(T1)**: `위험등급이 high인 고객 수는?` → 기대: `sql`, **"7명"**
2. **후속(T2)**: `그럼 그 등급 고객들은 어떤 세그먼트가 많아?`
   - **기대**: "그 등급"=high를 **맥락으로 이어받아** 세그먼트별 분포(private/sme/retail …) 답변
   - **말할 점**: "대화 메모리가 있어서 '그 등급'이 무엇인지 다시 말 안 해도 됩니다. 브라우저별 세션이라 사용자끼리 섞이지도 않습니다."
3. **🆕 새 대화** 클릭 → 화면 초기화 → 같은 후속 질문(`그럼 그 등급 고객들은…`) 다시 → **맥락 없이** 처리되는 것 대조
   - **말할 점**: "주제를 바꿀 땐 '새 대화'로 깨끗하게 리셋합니다."

> 무대 팁: 멀티턴은 후속 질문을 **짧고 명확히**. 주제가 바뀌면 반드시 "새 대화"로 리셋(이전 맥락이 끌려오는 것 방지).

## Act 3 ★ — No-code 즉석 적재 → 답 변화 → 삭제 → 원복 (하이라이트) · 약 3분

RAG의 본질("근거 있으면 정확, 없으면 환각")을 **클릭만으로** 보여주는 핵심 데모.

1. **적재 전 질문**: `클라우드 보안인증제(CSAP)는 무엇이고 어느 기관이 평가하나?`
   → ❌ **환각**(코퍼스에 없어 엉뚱한 기관/모호한 답). "근거가 없으면 이렇게 됩니다."
   - **팁(중요)**: 앞에 **"무엇이고"** 를 붙이세요. 이러면 에이전트가 **벡터 검색을 켜서** 적재 후 **새 문서를 근거로 인용**합니다. "…어느 기관이 평가하나?"만 물으면 관계형으로 읽혀 **graph로만 가서 인용이 안 붙을 수** 있습니다(답은 맞아도 "새 문서 인용" 효과가 사라짐).
2. **⚙ 코퍼스** 드로어 열기 → **텍스트** 탭 → 아래 내용 붙여넣기 → **＋ 추가 적재**

   - 제목: `클라우드 보안인증제(CSAP)`
   - 본문:
     ```
     클라우드 보안인증제(CSAP)는 클라우드 서비스의 보안성을 평가·인증하는 제도다.
     평가·인증 주체는 한국인터넷진흥원(KISA)이며, 인증 등급은 하·중·상 3등급으로 구분한다.
     공공기관에 클라우드 서비스를 공급하려면 CSAP 인증을 받아야 한다.
     ```
   → "적재 완료" 토스트, 문서 목록 8 → 9 (청크/엔티티/엣지 수 표시)
3. **같은 질문 재질의**(🆕 새 대화 후): `클라우드 보안인증제(CSAP)는 무엇이고 어느 기관이 평가하나?`
   → ✅ **정확**: "한국인터넷진흥원(KISA), 하·중·상 3등급" + **새 문서 인용**(vector 배지 점등 + CSAP 청크가 근거 패널에)
   - **말할 점**: "방금 붙여넣은 문서를 근거로, 인용까지 달아 정확히 답합니다. 코드 한 줄 없이요."
4. **🗑 삭제**(드로어 목록에서) → 같은 질문 → 다시 **환각으로 원복**
   - **말할 점**: "근거를 빼면 다시 못 맞춥니다. 'grounding이 정확도를 만든다'를 30초 만에 보여드린 겁니다."

입력 3종 모두 동작: **텍스트**(주력·가장 빠름) · **URL**(docling fetch) · **파일 PDF/DOCX**(docling 변환).
한 번의 적재로 **4개 저장소에 동시 반영**(OpenSearch·AstraDB KG·doc_registry·Iceberg, 같은 doc_id).

## Act 4 — 설명가능성 마무리 · 약 1분

직전 답변들의 근거 패널을 다시 펼쳐 짚는다.
- **tool-trace 배지** — 이 답에 어떤 도구(vector/graph/sql)를 썼는지
- **인용 칩** — 답변 주장 ↔ 출처 문서 연결
- **근거 패널** — 검색 청크(유사도) · KG 서브그래프(엔티티·엣지) · **생성 SQL + 결과 표**
- **출처 링크** — URL 문서는 원문, 직접작성 md는 `/corpus/*.md`로 열림
- **말할 점**: "답만 주는 게 아니라 '왜 그렇게 답했는지'를 항상 같이 보여줍니다. 규제 산업에서 신뢰의 조건이죠."

## Act 5 (선택) — Langflow로 오케스트레이션을 눈으로 · 약 2분

같은 도구를 **에이전트가 tool-calling으로 고르는 과정**을 시각적으로.
- Langflow: https://langflow-genai-apps.apps.itz-xe29ld.infra01-lb.tok04.techzone.ibm.com
- 플로우 **"Agentic RAG + KG (watsonx)"** 열기 → `ChatInput → Agent(watsonx granite) → ChatOutput` + 3 도구
  (`search_documents`/`lookup_relationships`/`query_business_data`)
- Playground에서 `위험등급 high 고객 수?` → Agent가 **query_business_data** 호출하는 단계가 보임 → "7명"
- **말할 점**: "챗 UI 뒤에서 실제로 이 에이전트가 돕니다. rag-ui가 도구를 HTTP로 노출하고, watsonx 에이전트가 질문에 맞는 도구를 직접 호출합니다."

## 백업 / 트러블슈팅

- **첫 응답 지연(콜드스타트)**: 사전 워밍업으로 예방. 무대에선 워밍업한 경로부터.
- **Langflow가 느리거나 실패**: 챗 UI는 자동으로 **파이썬 에이전트로 폴백**(단일턴). 데모는 계속 진행됨.
  강제 전환 필요 시: `oc -n genai-apps set env deploy/rag-ui RAG_ENGINE=python` → 롤아웃.
- **graph가 안 켜짐**: 질문에 "지식그래프/관계/엣지"를 명시.
- **text-to-SQL 0행/엉뚱**: 질문을 단순화(복잡 조인은 few-shot 범위). 예시 질문 그대로 쓰면 안전.
- **멀티턴이 엉킴**: "🆕 새 대화"로 리셋.
- **즉석 적재 후 답이 안 바뀜**: 적재 완료 토스트/목록 증가 확인, 새 대화로 재질의(이전 맥락 영향 배제).
- **답은 맞는데 인용(근거 청크)이 안 붙음**: 질문이 관계형으로 읽혀 graph로만 갔을 수 있음 → **"무엇이고…", "…에 대해 설명해줘"** 처럼 정의형으로 물어 **vector를 켜면** 문서가 인용됨.

## 데모용 안전 질문 모음 (검증됨)
- vector: `가명정보란 무엇이고 동의 없이 쓸 수 있는 목적은?`
- graph: `전자금융거래법의 접근매체와 감독기관의 관계는?`
- sql (AML 데이터셋): `위험등급이 high인 고객 수는?` / `위험등급이 high인 고객의 플래그된 거래 총액을 상대국가별로?` /
  `고객 세그먼트별 의심거래보고(STR) 건수와 신고금액 합계는?`
- sql (문서서 추출한 의무 테이블): `법별로 의무가 몇 건씩인지 많은 순으로?` / `의무 주체별로 의무 건수는?`
- vector (메타·자기설명): `Docling 문서 포맷은?` / `Langflow는 무엇이고 어떻게 쓰나` (URL 적재된 README 근거 · 짧은 키워드형이 안정적)
- 멀티턴: `위험등급이 high인 고객 수는?` → `그럼 그 등급 고객들은 어떤 세그먼트가 많아?`
- 즉석 적재: `클라우드 보안인증제(CSAP)는 무엇이고 어느 기관이 평가하나?` (적재 전 환각 → 후 정확+새 문서 인용 · "무엇이고"가 vector를 켜 인용 보장)
